**Frontend (Live):** https://smart-resource-allocation.vercel.app

## Architecture

```
Frontend (React/Vite) ──── Vercel (free, always on)
        ↕ REST API
Backend (FastAPI)      ──── Render (free tier)
        ↕ SQLAlchemy
Database (PostgreSQL)  ──── Supabase (free, always on)
```

---

## Environment Variables

### Backend (Render → Environment)

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | ✅ Yes | Supabase connection string (pooler format) |
| `SECRET_KEY` | ✅ Yes | JWT signing key — auto-generated by render.yaml |
| `FRONTEND_URL` | ✅ Yes | `https://smart-resource-allocation.vercel.app` |
| `ADMIN_EMAIL` | Optional | Admin login email (default: admin@smartalloc.org) |
| `ADMIN_PASSWORD` | ✅ Yes | Strong password — set this before first deploy |
| `ADMIN_FULL_NAME` | Optional | Admin display name (default: Platform Admin) |

**Get Supabase DATABASE_URL:**  
Supabase Dashboard → Settings → Database → Connection string → Shared Pooler  
Format: `postgresql://postgres.[ref]:[password]@aws-1-ap-south-1.pooler.supabase.com:5432/postgres`

### Frontend (Vercel → Settings → Environment Variables)

| Variable | Value |
|----------|-------|
| `VITE_API_URL` | `https://smart-resource-allocation-pvyg.onrender.com/api` |

## Deploy Backend (Render)

1. Go to https://dashboard.render.com
2. Find your `smart-resource-allocation` web service
3. Go to **Environment** → set all variables above
4. Click **Manual Deploy** → **Deploy latest commit**

The start command runs automatically:
```
python migrate.py && python seed_prod.py && uvicorn src.main:app --host 0.0.0.0 --port $PORT
```

- `migrate.py` — adds any missing DB columns (safe, idempotent)
- `seed_prod.py` — creates admin account if none exists (safe, idempotent)
- `uvicorn` — starts the API server

## Deploy Frontend (Vercel)

Vercel auto-deploys on every push to `main`. To trigger manually:
1. Go to https://vercel.com/dashboard
2. Find your project → **Deployments** → **Redeploy**

## First Login

After deployment:
1. Visit https://smart-resource-allocation.vercel.app
2. Log in with the admin credentials you set in `ADMIN_EMAIL` / `ADMIN_PASSWORD`
3. Go to **User Management** → create your team accounts
4. Go to **Community Needs** → add real needs from your area
5. Go to **Tasks** → create tasks linked to those needs

## Supabase Database Migration

If you need to run the schema migration manually:
1. Go to https://supabase.com/dashboard → your project → **SQL Editor**
2. Open `backend/supabase_migration.sql`
3. Copy and run it

## Local Development

```bash
# Backend
cd backend
python -m venv venv
.\venv\Scripts\activate        # Windows
pip install -r requirements.txt
cp .env.example .env           # fill in your Supabase DATABASE_URL
uvicorn src.main:app --reload
# API: http://localhost:8000/docs

# Frontend
cd frontend
npm install
npm run dev
# App: http://localhost:5173
```

## Troubleshooting

**Backend 500 errors?**
- Check Render logs for the exact error
- Run `python migrate.py` in Render Shell to apply missing columns
- Verify `DATABASE_URL` points to Supabase (not a Render-managed DB)

**Frontend can't connect to backend?**
- Check `VITE_API_URL` in Vercel environment variables
- Ensure `FRONTEND_URL` on Render matches your Vercel URL exactly
- Check browser console for CORS errors

**Render service sleeping?**
- Render free tier sleeps after 15 min of inactivity (50s cold start)
- Use UptimeRobot (free) to ping `/health` every 14 minutes
- Or upgrade to Render Starter ($7/month) for always-on

**Admin account not created?**
- Check Render logs for `seed_prod.py` output
- Run `python seed_prod.py` manually in Render Shell
